# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
아르바이트 일당 계산기




## 2. 시나리오 (5~10줄)
이 프로그램은 아르바이트생이 하루 근무를 마치고 자신의 일당을 간편하게 확인하기 위해 기획되었습니다. 사용자는 자신의 영문 이니셜, 시급, 근무 시간을 입력합니다. 프로그램은 입력된 시급과 시간을 곱하여 오늘 받을 총급여를 계산하고, 여기에 식비 등 추가로 받은 지원금을 더해 최종 일당을 출력합니다. 1차 단계에서는 단순 계산에 집중하며, 향후 주휴수당 판별(2차), 메뉴 시스템(3차), 월간 기록 관리(4차)로 확장할 예정입니다.

## 3. 예상 기능 및 메뉴 (최소 5개)

1.근무자 이니셜 입력 

2.시간당 급여(시급) 설정 

3.실제 근무 시간 입력 

4.식비 지원금 입력 

5.최종 합산 일당 출력 및 종료

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
-사용자로부터 알바생 이니셜, 시급, 근무 시간, 식비 지원금을 입력받아 최종 일당을 계산하여 출력합니다.
-캐릭터의 이니셜(문자), 시급/시간(정수), 식비(실수)를 입력받는 화면을 구현했습니다.
-사칙연산을 활용해 (시급 * 시간) + 식비 공식을 적용한 '최종 일당'을 계산하여 출력했습니다.
    
### **🤖 AI 파트너십 과정**
1. 내용 1


프롬프트 요약: "C언어 4단계 과제 로드맵(입출력 ➔ 조건문 ➔ 반복문 ➔ 배열/함수)에 적합한 아르바이트 급여 관리 시나리오 및 단계별 기능 확장 로드맵 기획 요청"
적용 내용: 단순한 계산기를 넘어, '기초 데이터 입력(1차) ➔ 주휴수당 및 세금 판별(2차) ➔ 메인 메뉴 시스템 구축(3차) ➔ 다수 인원 및 월간 기록 관리(4차)'로 이어지는 논리적인 시스템 확장 로드맵을 확립함.
      
2. 내용 2
프롬프트 요약: "1차 과제 필수 요건(변수 5개, 자료형 3개 이상)을 충족하기 위한 데이터 구조 모델링 논의"
적용 내용: 데이터를 특성에 맞춰 문자형(char - 이니셜), 정수형(int - 시급, 시간), 실수형(float - 식비 지원금)으로 자료형을 명확히 분리하여 입력 데이터의 구조적 완성도를 높임.
    
### **🛠️ Troubleshooting & 기술 회고:**
 1. 문제 1:
 
 
 현상: scanf로 이니셜(문자)을 입력받을 때, 이전 입력의 엔터키가 버퍼에 남아 다음 입력이 건너뛰어지는 현상이 발생함.

 해결: AI의 조언에 따라 scanf(" %c", &initial);와 같이 %c 앞에 공백을 추가하여 입력 버퍼를 비우는 방식으로 해결함.

2. 문제 2: 정수와 실수의 혼합 연산 처리

현상: 정수(급여)와 실수(식비)를 더할 때 결과값이 자동으로 실수형으로 변환되거나 데이터 손실이 우려됨.

해결: 최종 결과값인 totalPay를 정수형으로 선언하고, 계산 과정에서 실수를 더한 뒤 정수로 저장하여 소수점 이하를 자연스럽게 처리하는 방식을 적용함.
